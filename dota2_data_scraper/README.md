# Dota 2 Data Scraper

## Описание проекта
Этот проект предназначен для автоматизированного сбора и обработки данных из различных источников, связанных с игрой Dota 2. Используется Selenium для парсинга данных с сайта Dota2ProTracker и их последующей обработки с помощью Pandas.

## Структура проекта

```
dota2_data_scraper/
│-- main.py               # Главный скрипт для запуска
│-- requirements.txt      # Список зависимостей проекта
│-- config.py             # Конфигурации (пути, настройки)
│-- modules/              # Папка с модулями
│   │-- __init__.py        # Указывает, что это пакет
│   │-- scraper.py         # Сбор данных через Selenium и BeautifulSoup
│   │-- data_processor.py  # Обработка данных pandas
│   │-- steam_finder.py    # Поиск пути установки Steam
│   │-- autorun.py         # Создание автозапуска
│   └-- utils.py           # Вспомогательные функции
│-- scripts/               # Папка со скриптами для запуска
│   │-- run_scraper.py      # Скрипт для запуска парсинга
│   │-- update_config.py    # Скрипт для обновления конфигурации
│   └-- cleanup.py          # Скрипт для очистки временных файлов
│-- tests/                 # Папка для тестов
│   │-- __init__.py         # Указывает, что это пакет
│   └-- test_scraper.py     # Тестирование парсера
│-- README.md              # Описание проекта
│-- .gitignore              # Игнорируемые файлы в Git
```

## Установка

1. Клонирование репозитория:
   ```bash
   git clone https://github.com/yourusername/dota2_data_scraper.git
   cd dota2_data_scraper
   ```

2. Установка зависимостей:
   ```bash
   pip install -r requirements.txt
   ```

## Использование

Запуск основного скрипта для парсинга и обработки данных:
```bash
python main.py
```

Запуск парсинга отдельно:
```bash
python scripts/run_scraper.py
```

## Конфигурация

Основные настройки проекта хранятся в файле `config.py`, включая пути, параметры подключения и настройки парсинга.

## Зависимости

Для работы проекта необходимы следующие библиотеки:

- `selenium` — для работы с браузером
- `pandas` — для обработки данных
- `beautifulsoup4` — для парсинга HTML
- `requests` — для HTTP-запросов
- `fake-useragent` — для генерации случайных User-Agent

## Тестирование

Запуск тестов осуществляется командой:
```bash
pytest tests/
```

## Автозапуск

Для автоматического запуска программы при старте системы используется модуль `autorun.py`, который создаёт ярлык в автозагрузке Windows.
